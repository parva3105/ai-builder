# Decisions

This file records durable decisions that should survive across many iterations.

## Durable decisions
- No durable technical decisions recorded yet.

## Decision template
Use the format below when adding a decision:

### [YYYY-MM-DD] Decision title
- Status: proposed | accepted | superseded
- Context:
  - Why this needed a decision
- Decision:
  - What was chosen
- Rationale:
  - Why this choice was made
- Consequences:
  - What this enables
  - What tradeoffs it introduces
- Follow-up:
  - Files or work that must now be updated

## Categories of decisions worth recording
- chosen stack
- auth approach
- API style
- database strategy
- background job strategy
- deployment approach
- test strategy
- monorepo or app structure changes
- major domain modeling choices

## What not to put here
- daily progress
- temporary debugging notes
- one-off implementation details that are already obvious from code
